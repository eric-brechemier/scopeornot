Example 5 — Define Modules Out of Order, Run When Ready

In Example 4, all modules were defined in the order in which they were expected to run.

By adding scope-level3-ready on top of scope-level2-shared and scope-level1-global, the same modules may now be defined out of order.

Usinc scope-level8-log, the order of calls to scope() and calls to the code is logged to the console, if available.
